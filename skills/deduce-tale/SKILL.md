# 末法残土 — 第一人称剧情推演

你不是在「写」故事——你是一个末法残土的角色，在按这个世界的规则**活一遍**。剧情是你活出来的，不是编出来的。

> **核心区别**：`write-tale` 是文学写作（修士手记协议），`deduce-tale` 是逻辑推演。这里不关注「文笔优美」，关注「规则自洽」。

> ⚠️ **铁律：代码规则是推演引擎，不是叙事语言**
>
> 服务器代码中的数值（HP、spirit_qi、tick、grid_w、base_attack…）是给推演逻辑用的**内部约束**，**绝不出现在最终叙事中**。一旦故事里出现以下任何词汇，即为违规：
>
> - ❌ "HP 只剩 47"、"体力恢复到 80"、"真元池 60/80"
> - ❌ "破坏了一格方块"、"走了 200 格"、"坐标 (-3600, -2600)"
> - ❌ "spirit_qi = 0.3"、"danger_level 4"、"tick 计数"
> - ❌ "base_attack 12"、"耐久 180"、"grid_w 1 grid_h 2"
>
> **正确的转换方式**：
> - 「HP 只剩 5%」→ 「左肋的伤口还在渗血，视线已经开始发黑」
> - 「spirit_qi 0.05」→ 「这里的灵气稀薄到打坐一整天也恢复不了几丝真元」
> - 「走了 200 格」→ 「从初醒原走到青云残峰脚下，走了大半天」
> - 「base_attack 12 铁剑」→ 「手里这把铁剑是墟市上花十五枚骨币换的，刃口还算齐整」
> - 「骨币·五 ×3」→ 「袋子里只剩三枚骨币，铸得粗糙，真元封得少，花起来心疼」
>
> **记住：角色活在一个真实世界里，不是在玩 Minecraft。**
> 她不知道什么叫 HP，只知道伤口有多疼；不知道什么叫 tick，只知道等一个人等了多久。

## 参数解析

用户输入格式：`/deduce-tale <角色身份> <初始情境>`

| 参数 | 说明 | 示例 |
|------|------|------|
| 角色身份 | 境界 + 职业/身份 | `引气·守墟人`、`凝脉·兽核猎人`、`醒灵·炼丹学徒` |
| 初始情境 | 当前处境 + 目标 | `被困坍缩渊出口，锚石只剩一枚`、`墟市上发现一张假裂隙图` |

若用户只给方向不给完整参数，先补全再开始。

## 推演流程（四阶段）

### 阶段一：角色落位（不输出给用户，内化）

1. **读世界观**：
   ```
   Read docs/worldview.md — 聚焦与角色相关的章节
   ```
   必读章节：灵压环境（二）、修炼体系（三）、战斗系统（四）、经济与交易（九）、死亡与重生（十二）、世界地理（十三）。

2. **扫馆藏层一**：
   ```bash
   # 按角色关键词搜馆藏
   grep -rl '<关键词>' docs/library/*/
   # 批量读层一
   for f in <匹配文件>; do jq '{title, essence}' "$f"; done
   ```

3. **在脑内建立角色卡**：

   | 维度 | 内容 |
   |------|------|
   | 身份 | 职业/来路 |
   | 年龄 | 当前岁数 + 境界对应的预期余寿 |
   | 境界 | 醒灵/引气/凝脉/固元/通灵 |
   | 真元值 | 按境界估算（醒灵≤10 / 引气≤30 / 凝脉≤80 / 固元≤150） |
   | 经脉状态 | 通了哪几条？有无损伤？ |
   | 位置 | 六域中的哪里？馈赠区/死域/坍缩渊/墟市？ |
   | 灵压 | 当前区域灵压值（-N 到 +N 之间） |
   | 弱点 | 至少 1 个与末法规则绑定的致命弱点 |
   | 目标 | 短期要什么？长期要什么？ |
   | 资产 | 骨币、丹药、道具、裂隙图 |

   **寿命参考**：
   - 凡人寿元约 60-70 年
   - 醒灵/引气 ≈ 凡人寿元（修炼不延寿，只是强化体魄）
   - 凝脉：+10-15 年（经脉拓宽减缓衰老）
   - 固元：+20-30 年（真元凝核稳固生机）
   - 通灵：+40-60 年（感知天道，寿元显著延长）
   - 重伤、经脉断裂、长期负灵压暴露、服用禁药均**折寿**
   - 续命丹可延长 10 年，但永久折损真元上限

   **硬约束**：
   - 境界决定能力上限（引气境不能在负灵域撑超过三个时辰）
   - 灵压值决定可用的灵气（死域灵气=0，真元只出不进）
   - 角色不能知道「编剧才知道」的信息（不能预知哪条路有埋伏，除非有裂隙图）

### 阶段二：逐轮推演（大回合 + 小回合）

推演分为两种粒度，agent 根据情境**理性判断**用哪种：

| 回合类型 | 时间跨度 | 适用场景 | 推演粒度 |
|----------|---------|---------|---------|
| **大回合** | 数日～数月 | 闭关修炼、跨域旅行、灵田种植、墟市蹲守、养伤恢复 | 概要：一次推演覆盖整段时期的转折点 |
| **小回合** | 数息～数个时辰 | 战斗、妖兽遭遇、讨价还价、炼丹、突破、逃命 | 细密：一次推演只覆盖一个关键动作 |

**理性判断原则**：
- 跨域走路 → 大回合（三五天路上发生了什么？遇到了谁？）
- 遇妖兽开打 → 切入小回合（每一刀、每一步、每一口真元）
- 打完了继续赶路 → 切回大回合
- 闭关冲击境界 → 大回合（三个月经脉寸进，最后一天冲关 → 切小回合）
- 不可滥用小回合——买碗面不需要三百字的推演

#### 每轮必做四件事（大小回合通用）：

1. **结算寿命** — 时间流逝了多少？角色老了多少？余寿还剩几何？
2. **结算资源** — 真元消耗/恢复、骨币收支、物品消耗/过期、骨币真元半衰（13 天折半）
3. **推演事件** — 这个时间段内发生了什么关键转折？
4. **写入文件** — 将这轮的叙事段落追加到 `docs/tales/<标题>.md`（详见阶段三）

#### 大回合推演模板：

```
## 大回合 N：<主题>（时间跨度：X 天/月）

### 时间推进
- 流逝：<天数>
- 角色年龄：<新年龄>，余寿估算：<约 X 年>
- 骨币半衰结算：<哪些骨币过期/贬值>

### 当前状态
- 位置/路线：<从哪里到哪里>
- 灵压变化：<沿途灵压波动>
- 真元变化：<自然恢复/消耗净值>
- 资产变化：<消耗了什么，获得了什么>

### 关键事件（1-3 个转折点）
1. <发生了什么>
2. <又发生了什么>

### 新危机
<这一大回合结束后，角色面临的新问题>

### 是否切入小回合？
- 若有关键冲突（战斗/突破/谈判等）→ 下一轮切小回合
- 若无 → 继续大回合
```

#### 小回合推演模板：

```
## 小回合 N.m：<具体动作>

### 即时状态
- 位置：<精确位置>
- 真元：<当前值>
- 伤势：<具体伤口+严重度>

### 角色意图
<这一瞬间角色最想要什么？>

### 世界约束
1. <灵压规则约束>
2. <境界能力约束>
3. <战斗/身体约束（体力3.0/刀、部位伤害特效等）>
4. <天道注意力约束（若适用）>

### 可选行动
A. <选项A：代价与概率>
B. <选项B：代价与概率>

### 推演结果
<选择最符合角色性格+世界规则的一条路径，写出结果>
<结果必须包含：得失、新伤势、新真元值、下一步>
```

#### 推演硬规则（大小回合通用）：

1. **获利必付代价** — 找到丹药 → 被标记；突破境界 → 引来伪灵脉；救人 → 真元排异
2. **天道冷漠** — 天道从不主动帮人，只在灵气失衡时「纠正」（通常是让更多人死）
3. **信息不对称** — 角色不知道地图外的世界；NPC 可能在说谎；裂隙图可能画错
4. **资源守恒** — 骨币、丹药、真元用一点少一点，除非角色主动获取
5. **寿命是终极资源** — 时间不可逆，每一大回合都在消耗余寿；重伤、禁药、负灵压都在加速衰老
6. **每轮必出新危机** — 不能连续两轮平安无事（大回合的危机是方向性的，小回合的危机是即时的）

#### 推演结束条件：

- 角色死亡 → 写死亡场景，标注 `【推演终止：角色死亡】`
- 角色达成目标 → 写结局，标注 `【推演终止：目标达成】`
- 寿元耗尽（老死）→ 写临终场景，标注 `【推演终止：寿元耗尽】`
- 推演轮数 ≥ 15 且无明显收束趋势 → 强制收束，写出开放式结局

### 阶段三：每轮写入（不是最后才写）

**每完成一轮推演（无论大小回合），立即把这一轮的叙事追加到文件。**

#### 首次写入（第一轮结束后）：

```
Write docs/tales/<故事标题>.md
```

文件头：

```markdown
# <故事标题>

> 末法残土推演叙事 · 第一人称
> 角色：<身份> · <境界> · 年<岁数>
> 起始：<初始情境一句话>

---

<第一轮叙事>
```

#### 后续每轮追加：

```
Edit docs/tales/<故事标题>.md — 在文件末尾追加新段落
```

**叙事格式要求**：
- **纯文学段落** — 不分节标题、不做对话格式、不用 User/Assistant 标注
- **段落就是段落** — 用空行分段，用 `---` 分隔大的时间跳跃
- **第一人称流畅叙事** — 读者像在读一本日记/回忆录，不是在看 chat log
- **大回合叙事**：一两段概括长时间内的事件和心境变化
- **小回合叙事**：场景细腻，有具体的动作、对话、感官细节

#### 去游戏化转换（每轮叙事前自查）：

1. **推演逻辑不写入正文** — 推演中的「世界约束」「可选路径」是给 agent 看的，不出现在最终故事中
2. **第一人称** — 全部用「我」的视角，读者只能知道角色知道的事
3. **自然衔接** — 场景之间用时间跳跃留白（「第三天」「那之后两个月」）而非机械转场词
4. **末法意象收尾** — 最后一个场景用具体的末法道具/景象作结
5. **去游戏化** ⚠️ — 数值、坐标、游戏术语全部转换成感官体验和自然语言（详见顶部铁律）

#### 叙事风格（区别于 write-tale）：

- deduce-tale **不强制**修士手记协议的全部约束
- **但自然遵守**以下几条（因为推演本身就禁止这些）：
  - 人物不会直接说「我在乎你」（推演角色按逻辑行事，不按言情剧本）
  - 没有无代价的金手指（推演规则强制代价）
  - 没有功能性反派（推演中的 NPC 有自己的动机）
  - 没有机械降神（天道在推演中永远是冷漠的平衡机器）

### 阶段四：收束

推演终止后，在文件末尾写一段收束，然后关闭文件。

收束格式：

```markdown
---

*<终局标注：角色死亡 / 目标达成 / 寿元耗尽 / 强制收束>*
```

不再追加内容。

### 真元物理常量（qi_physics/constants.rs）

| 常量 | 值 | 说明 |
|------|-----|------|
| QI_DECAY_PER_BLOCK | 0.03/格 | 离体真元每格衰减 |
| QI_AMBIENT_EXCRETION_PER_SEC | 0.001/s | 末法环境分解速率 |
| QI_EXCRETION_BASE | 0.30 | 异体排斥基础率 |
| QI_HALFLIFE_REFERENCE_DAYS | 13 天 | 骨币真元半衰 |
| QI_TSY_DRAIN_FACTOR | 0.5 | 坍缩渊抽真元系数 |
| QI_TSY_DRAIN_NONLINEAR_EXPONENT | 1.5 | 坍缩渊非线性指数 (高境更惨) |
| QI_TSY_BASE_DRAIN_PER_TICK | 0.5/tick | TSY 基础抽取速率 |
| QI_CULTIVATION_REGEN_RATE | 0.01 | 修炼吸纳速率系数 |
| QI_ZONE_UNIT_CAPACITY | 50.0 | 1.0 zone 浓度可支撑的真元点数 |
| DEFAULT_SPIRIT_QI_TOTAL | 100.0 | 全服灵气预算默认值 |
| QI_MAINTENANCE_IDLE | 1.0 | 防御维持基线 |
| QI_DENSITY_GAZE_THRESHOLD | 0.85 | 灵物密度阈值 (触发天道关注) |

**推演换算公式**：
- 角色在 zone 中自然恢复真元 ≈ `zone.spirit_qi × QI_CULTIVATION_REGEN_RATE × qi_max` / tick
- 坍缩渊对引气(30)真元抽取 ≈ `0.5 × (30/100)^1.5 × 0.5` ≈ 0.04/tick (约 0.8/s)
- 坍缩渊对固元(150)真元抽取 ≈ `0.5 × (150/100)^1.5 × 0.5` ≈ 0.46/tick (约 9.2/s)

### 关键约束总结

推演时检查以下硬约束：

1. **真元 ≠ 无限** — 死域/负灵域真元只出不进，战斗体力 3.0/刀
2. **灵压 ≠ 均匀** — 六域 spirit_qi 不同 (0.05~0.7)，域间荒野 ≈ 0
3. **距离 ≠ 瞬移** — 域间 2000-5000 格，步行耗时以小时计
4. **骨币 ≠ 永恒** — 半衰 13 天，面值按真元含量分三档：小额(~0.3-1点)、中额(~2-3点)、大额(~5-8点)。游戏 item ID (5/15/40) 不是实际真元含量
5. **死亡 ≠ 结束** — 3 次命运，重生后虚弱 3 分钟
6. **真元排异** — 异体排斥基础率 30%，他人真元入体是毒
7. **灵物磨损** — 有灵之物转移必扣纯度，天道税不可免
8. **坍缩渊非线性** — 高境受抽真元更重 (指数 1.5)

## 不要做的事

- **不要**跳过世界观和代码查询 — 推演前必须读 worldview + 馆藏层一 + 本文件代码约束
- **不要**凭记忆编造数值 — 灵压、距离、物价必须对照 zones.json / constants.rs / items/*.toml
- **不要**让角色做出违反代码约束的行为 (引气在负灵域待一整天不伤、死域里释放外放术法)
- **不要**让推演变成「角色一直走好运」 — 每轮必须有代价
- **不要**用天道机械降神解决困境
- **不要**让角色拥有编剧视角的信息
- **不要**跳过推演直接编故事 — 推演逻辑是核心价值
- **不要**把推演日志写入最终故事 — 推演是过程，故事是结果
- **不要**使用 worldview 中不存在或代码未实现的物品/机制
- **不要**在叙事中出现游戏术语 — HP/体力/tick/坐标/spirit_qi/base_attack/grid/danger_level 等一律替换为感官描述
- **不要**把「一格」「一块」「一个方块」当作空间单位 — 用「几步」「半里」「一个时辰的路」
- **不要**在叙事里出现 User/Assistant 对话格式 — 这是小说，不是聊天记录
- **不要**只写小回合不写大回合 — 一场战斗之后，角色要吃饭睡觉赶路养伤，这些都需要大回合来推演
- **不要**忽略寿命 — 每轮大回合必须结算时间流逝和余寿变化
- **不要**等全部推演完才写文件 — 每轮结束立刻追加叙事到 `docs/tales/`
