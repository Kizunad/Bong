# Bong · plan-sou-da-che-v1

搜打撤循环节奏与风险节拍——末法残土的核心 gameplay loop 是**搜（探索/采集）→ 打（战斗/博弈）→ 撤（撤离/回家）**。现有系统层已齐备（botany 采集 ✅ / combat 战斗 ✅ / tsy 坍缩渊撤离 ✅ / niche 灵龛安全区 ✅），但循环之间的**张力曲线**未设计——玩家何时感到安全、何时紧张、何时必须做出撤退决策、两趟 run 之间的节奏如何起落。本 plan 不新增子系统，而是设计**风险节拍与情感曲线**，让搜打撤从"一堆可做的事"变成"有呼吸感的生存循环"。

> **2026-05-11 依赖实地核验通过**：全部 14 个前置 plan 已实装并注册在 `server/src/main.rs`。botany harvest（34 fn）/ combat resolve（58 fn，33 测试文件）/ tsy lifecycle（8 集成测试）/ niche-defense（已归档）/ qi_physics（field/ledger/channeling）/ death_lifecycle（BiographyEntry 29 fn）/ fauna（butcher/drop/rat_phase）/ jiezeq（Season enum，42 fn，20 测试）/ pseudo_vein（35 fn）/ identity（10 测试文件）/ style_telemetry（QiColor 追踪）— 无阻塞缺口。

**世界观锚点**：`worldview.md §十` "搜打撤"循环描述 · `§二` 灵压三态（馈赠区→死域→负灵域）的梯度风险 · `§四` 战斗是真元汇率兑换 · `§七` 生态联动（兽潮/负压灭杀）· `§十六` 坍缩渊内搜打撤的极致压缩

## §0 设计轴心

- [ ] **三态张力曲线**：每趟 run 经历"低压探索→中压博弈→高压撤离"的情感弧线，玩家在 run 结束时应感到"终于安全了"的释放
- [ ] **风险可读**：玩家不靠数字面板判断风险——靠环境线索（植被枯萎=灵气低 / 野兽密度↑=接近资源点 / 地面残灰=蜘蛛伏击区 / 远处雷声=天劫多发 / 天色突变=汐转期将至）
- [ ] **撤退决策窗口**：不是"血低了就跑"——是真元/背包容量/时间/节律/附近威胁 五个维度的复合决策。系统应提供足够的**决策信号**让玩家做出有意义的判断，而非靠猜
- [ ] **Run 间节奏**：一趟 run 结束后应有"回家—整理—规划—再出发"的呼吸感。不是做完就跑下一趟，而是每次回家都是一个小型的"整理战利品 + 查看骨币贬值 + 更新地图认知 + 选择下一次目标"的决策时刻
- [ ] **坍缩渊是搜打撤的放大器**：坍缩渊内三态压缩到极限——搜 = 扫容器、打 = 道伥+PvP、撤 = race-out。坍缩渊的设计已是完整闭环，本 plan 专注**主世界**搜打撤的节奏设计

## P0 — 风险梯度地图 ⬜

### 交付物

1. **四轴风险热力图层**（`server/src/world/risk_heatmap.rs`）
   - 不暴露给客户端——server 侧内部使用，驱动 NPC 行为 + 野兽 spawn + agent narration
   - **灵气轴**：灵气 0-0.1（死域）= 安全但无资源 / 0.3-0.5（馈赠区）= 中等风险（野兽+其他修士）/ 0.6+（灵眼/灵泉）= 高风险（高密度竞争）
   - **野兽密度轴**：噬元鼠密度 + 异变兽巡游路径 + 缝合兽领地范围
   - **NPC 活跃度轴**：散修密度 + 守墓人巡逻路线 + 枯骨休眠体分布
   - **玩家密度轴**（通灵+ 可见低境玩家轨迹——残灰脚印/打坐痕迹）
   - 四轴叠加 → 每个区块的 `RiskScore`（0-100）

2. **风险梯度对玩家行为的影响**
   - 低风险区（0-20）：采集效率正常，打坐安全，但资源稀少
   - 中风险区（20-50）：采集触发野兽概率↑，修炼可能被打断
   - 高风险区（50-80）：资源丰富但竞争激烈，战斗难以避免
   - 极端风险区（80-100）：灵眼/坍缩渊入口/伪灵脉中心——最高回报 + 最高死亡率
   - **关键设计**：不同境界对同一区域的 `RiskScore` 不同——通灵在馈赠区是食物链顶端（RiskScore -20），引气在同样位置是食物链底端（RiskScore +30）

3. **"搜"的路线设计**
   - spawn_plain → broken_peaks → spring_marsh → rift_valley → waste_plateau
   - 每条路线的 risk 渐变曲线（从安全→中等→高→安全——必须是 round trip）
   - 没有单向递增的路线——玩家总要回家

### 验收抓手

- 测试：`server::world::tests::risk_score_by_realm` / `server::world::tests::risk_heatmap_four_axis_overlay`
- 可视化：`/dev riskmap` 命令输出当前区块 risk 四轴数据（GM 调试用）

## P1 — 环境风险信号 ⬜

### 交付物

1. **植被风险信号**（复用 botany 系统）
   - 灵气 > 0.5 区域：植物茂密、颜色鲜艳（视觉安全但实际高风险——资源多人抢）
   - 灵气 0.1-0.2 区域：植物稀疏、颜色灰暗（视觉荒凉但实际中等安全——少人少兽）
   - 灵气 = 0（死域）：残灰方块 + 枯死树木 + 无植被（绝对安全的荒芜）
   - 负灵域边缘：噬灵藓蔓延 + 植物反向生长（向负压中心倾斜）→ 视觉警告
   - 曾被天劫劈过：焦黑地面 + 植物永久不生长 → 视觉警示"这里天道来过"

2. **音效风险信号**（依赖 `plan-audio-world-v1` 🆕 active）
   - 安全区音效：微风 + 偶尔鸟鸣 + 脚步回声正常
   - 中等风险音效：远处野兽低吼 + 偶尔树枝断裂 + 脚步声混响变闷（暗示附近有遮挡）
   - 高风险音效：噬元鼠尖啸 + 异变兽沉重脚步声 + 天道低声沉吟（`era_decree` whisper）+ 音效混响逐渐消失（"世界在安静——不是什么好事"）
   - 负灵域边缘音效：低频嗡鸣 + 真元流失的微妙"嘶嘶"声 + 心跳声逐渐清晰（真元越低心跳越响）

3. **粒子/VFX 风险信号**
   - 伪灵脉：地面微光粒子上升（金色，但闪烁不规则——暗示"这不是天然的"）
   - 异变兽领地：空气中偶尔飘过暗色孢子
   - 道伥出没区：地面偶尔浮现苍白人形残影，1s 消散
   - 汐转期将至：天空颜色渐变异常（夏→冬：暖橙→灰蓝 slow lerp）

4. **NPC 行为作为风险信号**
   - 散修突然蹲伏/靠墙 → 说明附近有威胁
   - 噬元鼠群突然向同一方向逃窜 → 反方向是危险区
   - 枯骨休眠体提前 5s 颤动 → 有高境修士靠近（负压或高阶）
   - 守墓人在某区域驻足停留 → 该区域灵气快被抽干了

### 验收抓手

- 手动：在不同风险区域之间穿行 → 闭眼只听音效能判断大致风险 → 睁眼看植被能判断灵气浓度 → 综合判断"这里该走还是该留"
- 测试：`server::world::tests::flora_density_by_qi` / `client::audio::tests::risk_ambient_transition`

## P2 — 撤退决策辅助 ⬜

### 交付物

1. **五维撤退 Checklist 可视化**（`client/src/main/java/com/bong/client/hud/ExtractDecisionHud.java`）

| 维度 | 信号 | 阈值提示 | 视觉 |
|------|------|---------|------|
| 真元 | 当前/上限 | < 30% → 暗红闪烁 | 真元条颜色渐变 |
| 背包 | 已用格/总格 | > 80% → 背包 icon 微胀 | 背包 icon 微动效 |
| 时间 | 在世本次 run 时长 | > 40min → "该回了"气泡 | 小沙漏 icon |
| 节律 | 当前汐转状态 | 汐转期 → "不稳"提示 | 天空边缘色变 |
| 威胁 | 附近 RiskScore | > 60 → 危险信号 | 迷你雷达红点 |

   - **关键设计**：不显示数字——用颜色/形状/动画传递。真元 < 30% = 条变暗红 + 微脉冲（不是"数字 12/40"）
   - checklist 纵向排列，每项单独 bad→good 渐变条
   - 退出决策不由 checklist 替玩家做——checklist 只是"你该看看这些了"

2. **"该回了"自然触发**
   - 不弹提示——通过环境变化暗示：
     - 天色渐暗（不管实际时间，run 超过 40min 后天空开始微暗）
     - NPC 气泡"天快黑了，还在外面？"
     - 噬元鼠群开始向巢穴方向移动（鼠群回家了 = 你也该回了）
     - 远处传来守墓人的低声吟诵（每晚一次，提示"夜晚的荒野不是你的地方"）
   - 玩家可以无视，继续跑——但夜间野兽密度 +50%，风险上升

3. **紧急撤退窗口**
   - 当 `RiskScore` 突然飙升（>80 且玩家境界不匹配）→ 触发"紧急撤退信号"
     - 附近的鸟全部飞走（粒子 flocking 动画）
     - 地面微震（camera micro-shake 0.1）
     - NPC 全部向安全方向狂奔
     - 玩家的脚步声自动变响（"你在跑，你在逃"）
   - 紧急窗口持续 10-15 秒 → 之后威胁抵达（异变兽/高境修士/天劫）

### 验收抓手

- 测试：`client::hud::tests::extract_checklist_color_by_threshold` / `client::hud::tests::checklist_no_numbers`
- 手动：在馈赠区采到背包快满+真元 25% → 天色暗 → checklist 全部黄/红 → 决定回家 → 路上遇到紧急撤退信号 → 狂奔 → 安全到家

## P3 — Run 间节奏 ⬜

### 交付物

1. **回家整理循环**（`client/src/main/java/com/bong/client/loop/HomeSequence.java`）
   - 踏入灵龛 5 格内 → 触发"回家感"：
     - 音效切换为安全区音效（炉火 crackle + 轻柔弦乐）
     - HUD 全部恢复常色（不再有红色警告）
     - 灵气条显示"灵龛——无灵气"
     - 小动画：角色坐下/靠墙 + 深呼吸（3s 自动播放，可打断）
   - 整理步骤（自然引导，不强制）：
     - 打开背包 → 看到本次 run 收获（物品带"NEW"角标 30s）
     - 分仓决策：哪些留灵龛/哪些带身上/哪些埋外面（天道密度阈值机制自然推动）
     - 查看骨币贬值情况（shelflife 系统）
     - 查看伤势：伤口 healProgress 条 + 经脉损伤状态
     - 决定：现在炼丹/锻器/灵田维护 or 立刻再出发？

2. **世界状态变化感知**
   - 每次回家时，agent 生成简短"世界变化摘要"（`HomeReturnNarration`）：
     - "你在外 42 分钟。灵泉湿地的灵气又薄了三分。"
     - "有人在你走后在青云残峰渡了劫。西边多了片焦土。"
     - "北荒方向传来一声低吼。是灵兽？是机缘？是坟墓？"
   - 摘要不显示在 HUD——通过 NPC 聊天 / 灵龛内"镜子"inspact 获得
   - 设计意图：每次回家都感觉"世界在我不在时继续运转"

3. **下一次目标选择**
   - 根据本次 run 收获 + 当前需求 + 世界状态 → agent 生成 2-3 个模糊建议（通过散修 NPC 气泡）
     - "听说灵泉湿地东边最近灵草疯长——但去的人都没回来。"
     - "你差一味赤髓草就能炼凝脉散了。血谷有，不过……"（顿一下）"那里最近不太平。"
   - 玩家自行决定——agent 只提供情报，不下达任务

### 验收抓手

- 手动：完成一趟 run → 回家 → 触发"回家感" → 整理战利品 → 听 NPC 聊世界变化 → 决定下一次目标 → 感觉"这趟值了"或"下一趟必须去 X"
- 测试：`server::agent::tests::home_return_narration_by_changes`

## P4 — 世界事件节奏注入 ⬜

### 交付物

1. **事件触发时机表**（`server/assets/world/event_rhythm.json`）
   - 伪灵脉：
     - 触发条件：某区域玩家密度 > 阈值 + 灵气消耗过快
     - 最佳插入点：玩家刚完成一趟 run 正在回家路上 → 突然前方升起伪灵脉（"要不要绕路去看看？"）
     - 持续时间：30-60 分钟 → 玩家有时间决定是否参与
   - 兽潮：
     - 触发条件：大区域灵气即将归零 + 汐转期
     - 最佳插入点：玩家在馈赠区深处采集时 → 地面微震 → 鼠群尖叫 → 反向狂奔（"跟着鼠群跑 or 逆着鼠群找死域宝藏？"）
   - 汐转期天象：
     - 触发条件：季节交界
     - 最佳插入点：玩家在灵龛整理时 → 天空颜色渐变 → NPC 气泡"天地又要变了……"→ 影响下一次 run 的路线规划（避开汐转高风险区）
   - 域崩（区域毁灭）：
     - 触发条件：区域灵气归零 + 无修士停留
     - 最佳插入点：玩家在安全区 → 远处轰鸣 + 天空变暗 + 全服 narration "某地化为死域"→ 玩家如果曾在那里建灵龛 → 灵龛迁移（rare 事件，整个生涯 1-2 次）

2. **事件的情感作用**
   - 伪灵脉：诱惑 + 猜疑（"天道在钓鱼"）
   - 兽潮：恐慌 + 机会（"逆流而上可能捡到便宜"）
   - 汐转：不安 + 谨慎（"先别出门"）
   - 域崩：震撼 + 失去（"我的灵龛……"）
   - 天劫广播：好奇 + 围观冲动（"去看看是谁在渡劫"）

### 验收抓手

- 手动：在搜打撤循环的不同阶段被事件打断 → 事件不感觉突兀 → 事件改变了本趟 run 的决策 → 事件结束后感觉"这才是末法残土"
- 测试：`server::world::tests::event_trigger_timing_by_player_loop_phase`

## P5 — 全流程压测 ⬜

### 交付物

1. **连续 5 趟 Run 的情感曲线实测**
   - Run 1（短途采集）：spawn_plain → 采灵草 → 回家 —— 低压，建立 baseline
   - Run 2（中距探索）：spawn_plain → broken_peaks → 遭遇散修 → 战斗 → 回家 —— 中压，体验战斗博弈
   - Run 3（高风险采集）：灵泉湿地深处 → 遭遇异变兽 → 伤员撤退 —— 高压，体验撤退决策
   - Run 4（世界事件打断）：正在采集 → 伪灵脉升起 → 选择去 or 不去 —— 测试决策点
   - Run 5（坍缩渊）：进入浅层坍缩渊 → 搜刮 → race-out —— 极限压力
   - 每趟 run 后测量：情感评分（1-10）/ 决策满意度 / 下次目标明确度

2. **节奏卡点检查**
   - 每趟 run 中"安全→紧张→安全"的过渡是否流畅
   - 撤退决策窗口是否提供了足够的信号
   - Run 间回家整理是否让人感到"充电"而非"麻烦"
