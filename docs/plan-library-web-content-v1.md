### 0.2 Astro Collection Schema

`library-web/src/content/config.ts` 定义字段（**必须**全部填写）：

```ts
{
  title: string,         // 书名，半文言风，如 "《封灵骨币考》"
  hall: string,          // 分馆中文名，如 "经济志"（需与 hallSlug 对应，但本 plan 五馆未含经济，用现有五馆之一）
  hallSlug: string,      // 分馆 slug，必须是：cultivation / geography / peoples / ecology / world 之一
  shelf: string,         // 书架编号/主题，如 "丙三·币轨残片"
  catalogId: string,     // 馆编目 ID，如 "经-0047"
  value: string,         // 藏值（文学化，如 "中等·参阅级"）
  rarity: string,        // 稀有度（"常见" / "稀见" / "孤本" / "禁品"）
  status: string,        // 保存状态（"良好" / "残损" / "焚痕" / "蠹蚀"）
  quote: string,         // 一句摘录（展示在目录），20-40 字，半文言
  anchor: string,        // 收录锚点：来自哪里（如 "春沼·荒庙残卷" / "青云峰外门典藏"）
  catalogDate: string,   // 入库日期（架空年号，如 "末法纪 三八七"）
  lastEdit: string,      // 最后整理日期
}
```

## 3. 写作风格约束

### 3.1 总体基调

**半文言半白话**，接近现代修仙网文但略重文学气。不要纯文言（晦涩），不要纯白话（扁平）。

**参考口径**：
- ✅ "末法三百七十年来，骨币日贬，昨日一枚可市米三斗，今日尚余二斗半，贩夫皆言'币已朽'。"
- ❌（太文言）"夫末法之世，币贬日甚，黎民苦之。"
- ❌（太白话）"末法时代骨币一直在贬值，老百姓很困扰。"

### 3.2 叙事视角

每册**选定一位撰书人**，用其视角写（增强真实感）：
- 散修（最常见）：实地见闻、带感情色彩
- 宗门长老：理论化、偏学究
- 末法遗老：追忆灵气鼎盛、悲凉
- 农夫/商贾：接地气、关心柴米油盐

### 3.3 禁止项

- ❌ 不要出现 "灵石"（worldview 唯一硬通货是**封灵骨币**）
- ❌ 不要出现"系统"、"面板"、"属性点"、"升级"等现代游戏词汇
- ❌ 不要编造与 worldview.md 矛盾的设定（例：说"炼气即可飞行"—— 实际飞行是化虚境才有）
- ❌ 不要在 quote 字段塞入超过 40 字的长句
- ❌ 不要在内容里出现作者自称 "AI" / "我是 Claude" / "由 LLM 生成" 等

### 3.4 每册结构建议

```markdown
---
title: "《封灵骨币考》"
hall: "众生谱"
hallSlug: "peoples"
shelf: "丙三·币轨残片"
catalogId: "众-0047"
value: "中等·参阅级"
rarity: "稀见"
status: "残损"
quote: "币已朽，人心未朽；市未尽，道未尽。"
anchor: "春沼荒庙残卷"
catalogDate: "末法纪 三八七"
lastEdit: "末法纪 三九二"
---

## 一、来历

（200-300 字，说背景）

## 二、内文

（正文 400-800 字，分 2-4 节）

## 三、按语

（撰书人短评 / 现状注记，100-200 字）
```

正文长度：**600-1200 字/册**（不计 frontmatter），目录质感优先于篇幅。
