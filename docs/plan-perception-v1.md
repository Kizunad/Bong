# Bong · plan-perception-v1 · 模板

**视野与感知系统专项**。定义 Bong 的双感知系统：**肉眼视觉**（受境界限制的清晰距离 + 雾化）与**神识感知**（worldview 已定义的境界世界感知能力，UI 形态采用屏幕边缘指示器）。

**核心理念**：
- **视觉受限是修仙感的基石** —— 醒灵境就是个普通人，远处全是雾
- **神识独立于视觉** —— 墙后/视觉外的感知，是境界提升的实质性奖励
- **两套系统并行**，不互相替代

**交叉引用**：
- `../worldview.md §境界`（6 阶境界 + 世界感知列，本 plan 严格对齐）
- `plan-vfx-v1.md` §3（HUD 叠色，与本系统协调：入定时视觉扩展、入魔时视觉收缩）
- `plan-iris-integration-v1.md`（可选 shader 真模糊层）
- `../plan-cultivation-v1.md`（境界系统对接）

## §1 与 worldview 对齐：神识感知逐阶定义

直接采用 worldview §境界 表的"世界感知"列：

| 阶 | 境界 | 神识感知（worldview 已定义） |
|----|------|----------------------------|
| 0 | 醒灵 | 能看到灵气浓淡（HUD 灵气条） |
| 1 | 引气 | 能感知 50 格内**生物气息**（含墙后） |
| 2 | 凝脉 | 能感知**区域灵气精确值** |
| 3 | 固元 | 能感知**其他修士的大致境界**（worldview §604） |
| 4 | 通灵 | 能感知**天道注意力**（危机预警） |
| 5 | 化虚 | 服务器内仅 1-2 人（神识近乎全图） |

**注意**：这不是视觉，是"内观/神识/灵感"层面的额外感知。墙后能感知，但不能"看见"。

## §2 视觉系统（worldview 未定义，本 plan 新增）

### 2.1 视觉距离表（提案）

| 阶 | 境界 | 清晰视距 | 模糊视距 | fog 起始 | fog 完全 |
|----|------|---------|---------|---------|---------|
| 0 | 醒灵 | 50m | 150m | 50m | 200m |
| 1 | 引气 | 70m | 200m | 70m | 250m |
| 2 | 凝脉 | 100m | 280m | 100m | 350m |
| 3 | 固元 | 150m | 400m | 150m | 500m |
| 4 | 通灵 | 300m | 800m | 300m | 1km |
| 5 | 化虚 | 800m | 几无限制 | 800m | 3km+ |

**设计依据**：
- **醒灵 = 普通人**：50m 看清表情、150m 看清人形、200m 全雾。对应现实人眼极限
- **化虚 = 天花板**：800m 清晰已经是天人之界，更远靠神识——保留修仙世界感
- **每阶涨幅 1.5-2 倍**：突破时玩家有"我能看见更远了"的实质感受
- **渐变模糊**：清晰→模糊→雾，不硬切

### 2.2 视觉环境调整因子

视觉距离不是只看境界，还有：

| 因子 | 影响 |
|------|------|
| 夜间 | × 0.3 |
| 暴雨/雪 | × 0.5 |
| 普通雾区 | × 0.4 |
| **负灵域**（worldview §负灵域）| **× 0.5 且按境界递减**（境界越高被压制越狠，对应"高灵压抽吸"） |
| 入定状态 | × 1.5 |
| 入魔状态 | × 0.5 |
| 顿悟瞬间（短暂） | × 3 |

**负灵域设计要点**：化虚境进入负灵域可能视觉降到醒灵水平，呼应 worldview "高境界被抽干" 的机制——视觉是被抽干的第一表征。

### 2.3 实现路径

**Phase 1（纯 vanilla，无 Iris 依赖）**：
- Mixin `BackgroundRenderer.applyFog` 改 fog 起始/结束距离
- 不改 vanilla render distance（玩家自己设的画质选项）
- 在 vanilla render distance 内做"伪视距"——超出清晰距离的渲染加 fog 渐隐

**Phase 2（可选，归 plan-iris-integration-v1）**：
- shader pack 加距离模糊 pass（按 depth buffer 模糊）
- 真"远处糊掉"效果，比 vanilla fog 更细腻

## §3 神识系统：屏幕边缘指示器（UI 形态 C）

### 3.1 形态规范

参考《死亡搁浅》的 BB 探测器、《地平线》的专注模式：

```
       ↑                    
   ╔═══════╗
   ║       ║  ←─ 屏幕边缘
   ║       ║
←  ║       ║  →  ← 视野外的感知目标 = 边缘指示器
   ║       ║
   ║       ║
   ╚═══════╝
       ↓
```

- 视野**外**的感知目标在屏幕**对应方向边缘**显示一个微光标记
- 视野**内**的目标不显示边缘指示器（避免遮挡）
- 距离越近指示器越大/越亮
- 不同感知类型用不同颜色/形状

### 3.2 指示器类型与境界对应

| 触发境界 | 指示器 | 视觉 | 含义 |
|---------|--------|------|------|
| 1 引气 | 生物气息 | 淡白色圆点 | 50m 内活物（人/兽） |
| 2 凝脉 | 灵气波动 | 蓝色波纹 | 区域灵气富集点 |
| 3 固元 | 修士境界 | 数字/段位符号 | 其他修士的境界段位 |
| 3 固元 | 同阶警示 | 黄色三角 | 接近的同阶/高阶修士 |
| 4 通灵 | 天道之眼 | 红色裂纹 | 天道注意力聚焦的方向 |
| 4 通灵 | 危机预警 | 闪烁红圈 | 即将发生的负面事件方向 |
| 5 化虚 | 全方位感知 | 多重叠加 | 几乎全图 |

### 3.3 不做什么（明确边界）

- ❌ **不做小地图/雷达圆盘**——破坏沉浸感
- ❌ **不在 3D 世界中标点**（不做 Highlight Outline / 透视标记）——保留视觉的纯粹
- ❌ **不显示具体名字/血量**——神识是"感"不是"读取"
- ❌ **不替代视觉**——视野内的目标必须靠肉眼看，神识只补充视野外

### 3.4 实现路径

- 客户端 `PerceptionHudRenderer`（接 `BongHudOrchestrator`）
- 每帧从 `BongClientState.perception` 读"感知到的目标列表"（位置 + 类型 + 强度）
- 计算每个目标相对玩家相机的方向 → 投影到屏幕边缘
- 视野内（投影在屏幕内）→ 不画
- 视野外 → 画对应类型的边缘指示器

## §4 服务端权威与同步

### 4.1 视觉距离

- server 根据玩家境界 + 环境因子计算 fog 距离
- 通过 `bong:server_data` 推送 `{ visual: { fog_start, fog_end, modifier } }`
- client 收到后调 `BongClientState.setVisualParams()`，Mixin 读这个值

### 4.2 神识感知列表

- server 每 N tick（默认 5 tick = 0.25s）按玩家境界扫描感知半径内目标
- 推送 `{ perception: [{ type, x, y, z, intensity }, ...] }`
- client 渲染边缘指示器
- **频率节流**：固元境感知半径大但 5 tick 一更新够用，化虚境单独配置

### 4.3 反作弊

- client 不可自行扩大 fog 距离（即使改设置文件）
- 因为 Mixin 直接覆写 vanilla fog 参数，server 不推送高境界数据时 fog 永远卡在醒灵档
- 客户端"渲染距离"画质设置仍可调，但 fog 强制盖住

## §5 与其他系统的协调

### 5.1 与 HUD 叠色（plan-vfx-v1 §3）

| 状态 | 视觉影响 | 叠色 |
|------|---------|------|
| 入定 | 视觉 × 1.5 | 整屏淡青 |
| 入魔 | 视觉 × 0.5 | 边缘黑雾 |
| 顿悟瞬间 | 视觉 × 3（数秒）| 全屏金光闪 |
| 天劫接近 | 视觉 × 0.7 | 整屏灰蓝压抑 |
| 中毒/寒毒 | 视觉 × 0.8 | 对应色调染色 |

### 5.2 与战斗系统

- 低境界打高境界："对方在我视野外但我在对方视野内" → 自然劣势
- 弱者诱强者入负灵域：高境界视觉骤降到醒灵水平，盲战 → 反杀机会（worldview §55 "负压战术"的视觉化实现）
- 神识 + 隐匿：worldview 提到"隐匿"机制，需要 plan-stealth 配合（隐匿玩家不出现在神识列表）

### 5.3 与 worldview §602 "外观由境界段位决定"

- 视觉看到的"人形外观"由对方境界段决定（醒灵到化虚穿着不同）
- **但视觉不精确分辨境界**（引气和凝脉看起来差不多 —— worldview 原话）
- **要看本质必须靠神识**（固元境的"看到对方大致境界段"是神识能力，不是视觉）

## §7 已知风险

- **服务端 push 通道未铺**：Phase 0 是所有 Phase 的前置，必须先解决
- **Mixin BackgroundRenderer 与 OptiFine 兼容**：OptiFine 自己改 fog，可能冲突。Iris 不改 fog，安全
- **边缘指示器视觉污染**：感知目标过多时屏幕边缘会被指示器塞满，需要**优先级 + 数量上限**（默认每方向最多 3 个）
- **神识/视觉混淆**：玩家可能觉得"为什么我看到了又看不到" → 需要**入门引导**或 inspect 界面解释
- **化虚境性能**：感知半径几公里 + 全图扫描频繁，server 计算压力大，需要空间索引（chunk-based）
- **境界跌落的视觉冲击**：从固元降回凝脉，视觉骤缩 50m → 玩家可能不适，需要平滑过渡（5-10 秒淡入）
