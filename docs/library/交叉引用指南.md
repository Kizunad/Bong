# 馆藏交叉引用指南

> 如何保证每一卷新条目都与 worldview 和其他馆藏对齐，而不是各写各的。

---

## 问题

馆藏分五个分馆，各有各的 agent。如果不做约束，很容易出现：
- 地理志写了一种草，生态录完全不知道
- 修行藏的丹方需要某种材料，但生态录里没有这种材料
- 两个条目对同一个数值（灵气、坐标、刷新周期）写了不同的值
- worldview 改了设定，馆藏条目没跟着改

---

## 交叉引用的三条规则

### 1. 锚点链必须可追溯

每个关键数值或设定结论都要能追回到源头：

```
worldview.md（第一锚点）
  ↓ 引用
馆藏条目 A（第二锚点）
  ↓ 交叉引用
馆藏条目 B
```

**做法**：
- 条目的编目信息里写 `锚点来源`，指向 worldview 的具体章节
- 正文中引用其他条目时，用相对链接：`详见[《血谷残志》](../geography/血谷残志.md)`
- 如果一个数值来自另一个条目而非 worldview，要写明："赤髓草生长灵气阈值 0.25-0.35（见《血谷残志》子区域表）"

**反例**：直接写"灵气 0.3"却不标来源——如果将来 worldview 改了，没人知道要来改这里。

### 2. 同一事实只写一处，其余交叉引用

当一个主题横跨多个分馆时：

| 情况 | 主归属 | 其他分馆怎么写 |
|------|--------|---------------|
| 赤髓草的**生长区域和灵气条件** | 生态录（植物条目） | 地理志写"产出赤髓草（详见生态录）" |
| 赤髓草的**产地地形和采集路线** | 地理志（区域条目） | 生态录写"产地详见《血谷残志》" |
| 赤髓草**入丹配方和药效** | 修行藏（丹方条目）或生态录（药材条目） | 看主要讨论的是"怎么炼"还是"草本身" |
| 缝合兽的**生态习性** | 生态录 | 地理志写密度和分布，不重复写行为逻辑 |
| 缝合兽的**兽核用途** | 修行藏 | 生态录写"兽核详见修行藏" |

**判定主归属的方法**：问自己"这段话修改后，哪个分馆的 agent 最应该知道？"那就是主归属。

### 3. 数值对齐检查

以下数值在多个条目中反复出现，最容易出现不一致：

| 数值类型 | 权威来源 | 常见引用方 |
|---------|---------|-----------|
| 区域坐标、范围、灵气基准 | worldview.md 十二节 | 地理志所有区域条目 |
| 境界名称、突破条件、材料需求 | worldview.md 三节 | 修行藏、生态录（药材用途） |
| 野兽种类、行为特征 | worldview.md 七节 | 生态录、地理志（密度分布） |
| NPC 类型与交互规则 | worldview.md 十节 | 众生谱、生态录（炼丹疯子） |
| 灵气衰减公式、守恒规则 | worldview.md 二节 | 所有涉及灵气的条目 |

**做法**：写新条目时，先 grep worldview 中的相关数值，确认自己写的和源头一致。如果觉得源头的数值不合理（比如 worldview 表里的路程时间算错了），在条目中标注 `勘误备注`，不要擅自改 worldview。

---

## 写新条目的检查流程

```
1. 确定主归属分馆
2. grep worldview.md 中所有相关锚点，记下章节号
3. grep 已有馆藏中是否有重叠主题
   - 有重叠 → 确认主归属，在新条目中用交叉引用而非重复抄写
   - 有数值 → 对齐，发现不一致要标注
4. 写条目，每个关键结论标锚点来源
5. 无依据的推演标"本卷推演"，超出地理志/生态录职能范围的标"提案"
6. 写完后检查实现挂钩中的交叉引用链接是否正确
7. catalog-book.sh 收录
```

## 实际案例

### 案例：赤髓草的交叉引用网络

```
worldview.md (三、修炼体系)
  "凝脉散需要 3 种灵草，配方不固定"
    ↓
《辛草试毒录》(ecology-0001) — 主条目
  定义赤髓草的生长条件、辛度、药用、口感
  引用产地 → 《血谷残志》
  引用丹方 → worldview 三节 + 本卷丹方交叉表
    ↓                          ↓
《血谷残志》(geography-0001)    《爆脉流正法》(cultivation-0001)
  赤髓沟产出赤髓草（不重复药性）   经脉修复需要噬脉根（引用生态录）
  缝合兽守护采集点（不重复习性）
```

一条设定（赤髓草），三个条目各写各的职能内容，通过交叉引用串起来。改任何一处，顺着引用链就能知道其他哪些地方需要检查。

## 常见陷阱

1. **复制粘贴整段描述到另一个条目** — 改了一处忘了另一处。用交叉引用代替。
2. **"差不多就行"的数值** — 血谷灵气你写 0.3，他写 0.28，读者不知道谁对。统一引用 worldview。
3. **跨分馆的大设定悄悄塞进小条目** — 比如在地理志里顺手写了个"终结世界的封印"。超出本分馆职能的设定要标 `提案` 并指向对应分馆确认。
4. **只在正文里提了引用但实现挂钩没写** — 实现挂钩的交叉引用栏是给开发者看的，正文里的引用是给读者看的，两边都要有。
5. **worldview 更新后馆藏没跟着更新** — 这个目前靠人工。未来可考虑脚本扫描馆藏中引用的 worldview 章节号，检测 worldview 变更后自动标记受影响条目。
