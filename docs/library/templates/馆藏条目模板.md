# 《馆藏条目模板》

> “凡可立卷者，先定编目；编目不定，后来者无从索引。”

---

## 用途

这不是正式馆藏，而是**末法残土图书馆**的标准条目模板。

当你要新增一卷世界设定时，优先复制这份模板到对应分馆：

- `docs/library/world/`
- `docs/library/geography/`
- `docs/library/peoples/`
- `docs/library/ecology/`
- `docs/library/cultivation/`

写完后运行：

```bash
bash scripts/catalog-book.sh "docs/library/<分馆>/<书名>.md"
```

如果某个实现项已经在 server / client / agent / docs 中落地，再运行：

```bash
bash scripts/mark-implemented.sh "docs/library/<分馆>/<书名>.md" "服务端机制已实现"
```

## 编目说明

- **分馆**只能是：世界总志 / 地理志 / 众生谱 / 生态录 / 修行藏
- **书架**用于更细粒度分类，例如：总序、海图、药材典录、风味指南、流派抄本
- **藏书编号**建议用英文 slug，方便后续脚本与实现挂钩，例如：`ecology-herb-0003`
- **估值**直接写成“1骨币 / 3骨币 / 10骨币”这类可读格式
- **收录状态**在正式运行收录脚本前保持“待收录”，收录后脚本会改成“在架”
- **收录时间**与**最后整理**由脚本维护，不要手改成花哨格式
